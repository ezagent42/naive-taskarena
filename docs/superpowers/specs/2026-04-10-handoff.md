## 今天完成的工作

### 实例锁（防止多进程重复发早晨提醒）

**背景：** 发现残留的 `taskarena channel` 进程导致每天收到多条重复早晨提醒（每个进程独立发一条）。

**修复：**
- `channel.py` 新增 `acquire_instance_lock(lock_path)`：用 `fcntl.flock` 获取 `~/.taskarena.lock` 排他锁
- 已有实例运行时第二个进程立即 `exit(1)` 并打印提示
- `main()` 在 `asyncio.run()` 前调用，锁随进程退出自动释放
- 新增 `tests/test_channel_lock.py`（3 个测试）

**残留进程的根因：**
- 用 `uv run taskarena channel` 直接启动时，stdin 是终端，永远不会关闭，进程不会自动退出
- tmux 被强制关闭时 Claude 没有机会清理 child process

## 飞书 API 易错点

| 场景 | 正确做法 |
|------|---------|
| PATCH 任务字段 | 必须用 `PatchTaskRequestBody.builder().task(...).update_fields([...])` |
| 修改 assignee | 不能用 PATCH，必须用 `aadd_members`（独立接口） |
| `Member.type` | 必须是 `"user"`，不是 `"open_id"` |
| `Due` / `Start` timestamp | 毫秒字符串，`is_all_day=True` |
| `complete_task` | 设 `completed_at` 为当前毫秒时间戳字符串 |
| WebSocket 连接 | 同一 App 只允许一个连接，多进程互相抢占（已用实例锁解决） |
| 日志文件 | `.taskarena/taskarena.log`（DEBUG 级别） |
| 群消息 chat_type | 飞书 SDK 可能返回 `"group"` 或 `"group_chat"`，events.py 里用 `in ("group", "group_chat")` 判断 |
