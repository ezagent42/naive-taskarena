### Bug Fixes（已 commit，在 PR #4 中）

| Bug | 根因 | 修复 |
|-----|------|------|
| taskarena channel 进程退出后残留 | `stop_event` 只监听 SIGTERM，stdin 关闭时 `server.run` 正常返回但其他 task 不停 | `_run_server()` 包装函数，`server.run` 返回后自动 `stop_event.set()` |
| `update_task` / `complete_task` 报错 99992402 | `update_fields` 是 `PatchTaskRequestBody` 的字段，不是 query param | 用 `PatchTaskRequestBody.builder().task(...).update_fields([...])` |
| 认领任务失败 1470400 | `members` 不在 PATCH 的 `update_fields` 支持列表里 | 新增独立工具 `assign_task`，调用 `aadd_members` 专用接口；`Member.type` 必须是 `"user"` |
| 查询"我的任务"返回所有人的任务 | `list_tasks` 没有 assignee 过滤 | 加 `assignee_id` 参数，按 open_id 过滤 |
| 早晨提醒后 Claude 没有 task_id 上下文 | scheduler 直接发飞书私信，Claude 不知情 | scheduler 发完后通过 `notifier` 发 `morning_reminder_sent` 通知 |
| YAML `9:00` 无引号被解析为 int 540 | PyYAML sexagesimal | `str(cfg.morning_time).split(":")` + 捕获 `AttributeError` |

## 飞书 API 易错点（血泪教训）

| 场景 | 正确做法 |
|------|---------|
| PATCH 任务字段 | 必须用 `PatchTaskRequestBody.builder().task(...).update_fields([...])` |
| 修改 assignee | 不能用 PATCH，必须用 `aadd_members`（独立接口） |
| `Member.type` | 必须是 `"user"`，不是 `"open_id"` |
| `Due` / `Start` timestamp | 毫秒字符串，`is_all_day=True` |
| `complete_task` | 设 `completed_at` 为当前毫秒时间戳字符串 |
| WebSocket 连接 | 同一 App 只允许一个连接，多进程互相抢占 |
| 日志文件 | `.taskarena/taskarena.log`（DEBUG 级别） |
